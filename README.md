## 1. API (Interface de Programação de Aplicações)

**API (Interface de Programação de Aplicações)** é um conjunto de regras e definições que permite a comunicação entre diferentes softwares. Em desenvolvimento web e mobile, APIs são frequentemente usadas para integrar serviços externos ou acessar dados remotos.

## 2. Framework

Um **Framework** é uma estrutura de suporte que fornece abstrações e ferramentas padronizadas para o desenvolvimento de software. Ele ajuda os desenvolvedores a criar aplicativos mais eficientes e consistentes ao seguir um conjunto comum de convenções.

## 3. IDE (Ambiente de Desenvolvimento Integrado)

Um **IDE (Ambiente de Desenvolvimento Integrado)** é uma ferramenta que reúne vários recursos para facilitar o desenvolvimento de software. Ele geralmente inclui um editor de código, depurador e ferramentas de compilação, proporcionando um ambiente unificado para programadores.

## 4. Git

**Git** é um sistema de controle de versão amplamente utilizado para rastrear alterações em projetos de software. Ele permite que equipes colaborem de maneira eficiente, mantendo um histórico completo das alterações feitas no código.

## 5. Pull Request

Um **Pull Request** é uma solicitação feita por um desenvolvedor para incorporar suas alterações a um repositório. É comumente usado em sistemas de controle de versão distribuídos, como o Git, para revisão de código e integração de alterações.

## 6. Commit

Um **Commit** é uma ação que salva as alterações em um repositório Git. Cada commit tem um mensagem que descreve as alterações realizadas, proporcionando um histórico detalhado do desenvolvimento do código.

## 7. Branch

Uma **Branch** é uma linha de desenvolvimento independente em um repositório Git. Ela permite que os desenvolvedores trabalhem em recursos ou correções de bugs sem afetar o código principal. As branches podem ser fundidas posteriormente.

## 8. Merge

**Merge** é o processo de combinar alterações de uma branch para outra no Git. Isso é frequentemente usado para incorporar o trabalho concluído em uma branch de desenvolvimento principal.

## 9. Frontend

**Frontend** refere-se à parte de um aplicativo ou site que os usuários interagem diretamente. Desenvolvedores frontend lidam com a interface do usuário, design e interatividade.

## 10. Backend

**Backend** é a parte de um sistema que lida com a lógica de negócios, processamento de dados e interações com o banco de dados. Desenvolvedores backend se concentram na lógica que acontece nos bastidores.

## 11. HTML (Hypertext Markup Language)

**HTML (Hypertext Markup Language)** é uma linguagem de marcação usada para estruturar o conteúdo em uma página da web. Ele define elementos e tags para representar diferentes tipos de conteúdo.

## 12. CSS (Cascading Style Sheets)

**CSS (Cascading Style Sheets)** é uma linguagem de estilo usada para controlar a aparência e o layout de elementos HTML em uma página da web. Ele define estilos, como cores e fontes.

## 13. JavaScript

**JavaScript** é uma linguagem de programação amplamente usada para criar interatividade em páginas da web. Ele é executado no navegador do usuário e permite manipular o DOM, responder a eventos e realizar cálculos.

## 14. Variable (Variável)

Uma **Variável** é um espaço de armazenamento nomeado que contém um valor. Em programação, as variáveis são usadas para armazenar e manipular dados.

## 15. Function (Função)

Uma **Função** é um bloco de código que realiza uma tarefa específica. Ela pode receber parâmetros, processá-los e retornar um resultado. Funções são fundamentais para organizar e reutilizar código.

## 16. Array

Um **Array** é uma estrutura de dados que armazena uma coleção de elementos. Cada elemento é acessado por um índice, permitindo o armazenamento e acesso eficiente de dados.

## 17. Object (Objeto)

Um **Objeto** é uma estrutura de dados que contém propriedades, cada uma associada a um valor. É usado para representar entidades com características e comportamentos no código.

## 18. JSON (JavaScript Object Notation)

**JSON (JavaScript Object Notation)** é um formato leve de troca de dados. É comumente usado para transmitir dados entre um servidor e um cliente, sendo fácil de ler e escrever para humanos e máquinas.

## 19. REST (Representational State Transfer)

**REST (Representational State Transfer)** é um estilo arquitetural usado na construção de serviços web. Ele utiliza operações HTTP padrão (GET, POST, PUT, DELETE) para criar, ler, atualizar e excluir recursos.

## 20. Endpoint

**Endpoint** é um ponto final de uma API, representando um recurso específico. Os endpoints são URLs que podem ser acessados para interagir com os dados por meio de operações REST.

## 21. SQL (Structured Query Language)

**SQL (Structured Query Language)** é uma linguagem de programação usada para gerenciar e manipular bancos de dados relacionais. Ela permite a criação, leitura, atualização e exclusão de dados.

## 22. NoSQL

**NoSQL** é um termo geral para bancos de dados que não seguem o modelo relacional tradicional. Esses bancos de dados são flexíveis e adequados para dados não estruturados ou semiestruturados.

## 23. Middleware

**Middleware** é um software que atua como intermediário entre diferentes sistemas ou componentes. Ele facilita a comunicação e o gerenciamento de dados entre aplicativos.

## 24. Scrum

**Scrum** é um framework ágil para gerenciamento de projetos. Ele enfatiza a colaboração, transparência e entrega incremental, dividindo o trabalho em iterações chamadas sprints.

## 25. Dependency Injection

**Dependency Injection** é um padrão de design em que as dependências de um objeto são injetadas nele, em vez de serem criadas internamente. Isso melhora a modularidade e a testabilidade do código.

## 26. MVC (Model-View-Controller)

**MVC (Model-View-Controller)** é um padrão arquitetural que divide uma aplicação em três componentes principais: Modelo (lógica de negócios), Visão (interface do usuário) e Controlador (gerenciamento de fluxo).

## 27. Docker

**Docker** é uma plataforma de contêinerização que simplifica a implantação de aplicativos em ambientes isolados. Os contêineres Docker incluem tudo o que é necessário para executar um aplicativo.

## 28. Microservices

**Microservices** é uma abordagem arquitetural que projeta uma aplicação como um conjunto de serviços independentes, cada um executando uma função específica. Isso facilita a escalabilidade e a manutenção.

## 29. DevOps

**DevOps** é uma cultura que promove a colaboração entre desenvolvimento (Dev) e operações (Ops). O objetivo é acelerar o desenvolvimento e a entrega de software por meio da automação e integração contínua.

## 30. Continuous Integration (Integração Contínua)

**Continuous Integration (Integração Contínua)** é uma prática que envolve a integração frequente de alterações de código no repositório compartilhado. Isso ajuda a detectar e corrigir problemas rapidamente.

## 31. TDD (Test-Driven Development)

**TDD (Test-Driven Development)** é uma abordagem de desenvolvimento em que os testes são escritos antes do código de produção. Isso promove a qualidade do código e orienta o design do software.

## 32. Continuous Deployment (Implantação Contínua)

**Continuous Deployment (Implantação Contínua)** é a prática de liberar automaticamente novas versões de software para o ambiente de produção após passar por testes automatizados. Isso acelera a entrega de software.

## 33. JWT (JSON Web Token)

**JWT (JSON Web Token)** é um padrão aberto que define uma maneira compacta e autossuficiente de transmitir informações entre partes como um objeto JSON. É comumente usado para autenticação e autorização.

## 34. Scalability (Escalabilidade)

**Scalability (Escalabilidade)** refere-se à capacidade de um sistema lidar com um aumento crescente de carga ou demanda. Isso pode ser alcançado por meio de técnicas como balanceamento de carga e escalonamento horizontal.

## 35. Agile

**Agile** é um conjunto de valores e princípios que enfatizam a entrega rápida e flexível de software. Metodologias ágeis, como Scrum e Kanban, são amplamente usadas no desenvolvimento de software.

## 36. Algorithm (Algoritmo)

Um **Algoritmo** é um conjunto de passos ou regras definidos para realizar uma tarefa ou resolver um problema. Eles são fundamentais para o desenvolvimento de software, influenciando a eficiência e a complexidade dos programas.

## 37. Repository (Repositório)

Um **Repositório** é um local centralizado para armazenar e gerenciar versões de um projeto de software. É comumente usado em sistemas de controle de versão, como Git.

## 38. Code Review (Revisão de Código)

**Code Review (Revisão de Código)** é o processo de revisar e avaliar o código fonte de um colega desenvolvedor. Isso promove a qualidade do código, identifica problemas e compartilha conhecimento.

## 39. Linting

**Linting** é o processo de analisar o código em busca de erros, padrões não conformes e práticas questionáveis. Ferramentas de linting ajudam a manter um código limpo e consistente.

## 40. GraphQL

**GraphQL** é uma linguagem de consulta para APIs que permite aos clientes solicitar dados específicos. Diferentemente das APIs REST tradicionais, GraphQL permite que os clientes especifiquem a estrutura dos dados de que precisam.

## 41. WebSocket

**WebSocket** é um protocolo de comunicação bidirecional que permite a transferência de dados entre um cliente e um servidor em tempo real. Ele é frequentemente usado para aplicativos que exigem atualizações instantâneas.

## 42. Containerization

**Containerization** é o processo de empacotar um aplicativo e suas dependências em um contêiner isolado. Isso garante que o aplicativo seja consistente em diferentes ambientes.

## 43. ORM (Object-Relational Mapping)

**ORM (Object-Relational Mapping)** é uma técnica que mapeia objetos de um sistema orientado a objetos para tabelas de um banco de dados relacional. Facilita a interação entre o código e o banco de dados.

## 44. Singleton

**Singleton** é um padrão de design que garante a existência de apenas uma instância de uma classe em um programa. Isso é útil quando exatamente uma instância de uma classe é necessária para coordenar ações em todo o sistema.

## 45. Callback

Um **Callback** é uma função que é passada como argumento para outra função. Ela será executada posteriormente, muitas vezes após a conclusão de uma operação assíncrona.

## 46. DSL (Domain-Specific Language)

**DSL (Domain-Specific Language)** é uma linguagem de programação ou configuração dedicada a um domínio específico. É projetada para ser compreensível por pessoas que trabalham naquele domínio em particular.

## 47. Responsive Design (Design Responsivo)

**Responsive Design (Design Responsivo)** é uma abordagem de design que torna as interfaces de usuário adaptáveis a diferentes tamanhos de tela e dispositivos, proporcionando uma experiência consistente.

## 48. JWT (Just World Theory)

**JWT (Just World Theory)** é uma teoria psicológica e social que sugere que as pessoas acreditam que o mundo é um lugar justo, onde as ações são recompensadas ou punidas de acordo com sua moralidade.

## 49. Immutable

**Immutable** refere-se a objetos cujo estado não pode ser alterado após a criação. Isso ajuda a evitar efeitos colaterais indesejados e facilita o rastreamento de mudanças.

## 50. Webhooks

**Webhooks** são mecanismos que permitem que aplicativos enviem automaticamente informações para outros aplicativos quando ocorre um evento específico. Eles são frequentemente usados em integrações entre sistemas.

## 51. MVP (Minimum Viable Product)

**MVP (Minimum Viable Product)** é a versão mais básica de um produto que inclui apenas as funcionalidades essenciais. É lançado inicialmente para coletar feedback dos usuários.

## 52. Refactoring (Refatoração)

**Refactoring (Refatoração)** é o processo de reestruturar o código-fonte de um software sem alterar seu comportamento externo. Isso melhora a legibilidade, manutenção e eficiência.

## 53. JWT (Java Web Toolkit)

**JWT (Java Web Toolkit)** é uma biblioteca para criar aplicativos da web Java eficientes e escaláveis. Ele fornece ferramentas e estruturas para simplificar o desenvolvimento.

## 54. CDN (Content Delivery Network)

**CDN (Content Delivery Network)** é uma rede de servidores distribuídos globalmente que armazena cópias de conteúdo web. Isso melhora a velocidade de carregamento de páginas para usuários em diferentes regiões.

## 55. CORS (Cross-Origin Resource Sharing)

**CORS (Cross-Origin Resource Sharing)** é uma política de segurança que permite ou restringe o acesso a recursos em uma página web a partir de outros domínios. Ele evita ataques de solicitações cruzadas.

## 56. Regex (Expressão Regular)

**Regex (Expressão Regular)** é uma sequência de caracteres que forma um padrão de pesquisa. É amplamente usado para verificar e manipular strings, facilitando a busca por padrões específicos.

## 57. Serverless

**Serverless** é uma arquitetura de computação em que os desenvolvedores não precisam gerenciar servidores. As funções são executadas em resposta a eventos, e a infraestrutura é gerenciada automaticamente.

## 58. CRUD (Create, Read, Update, Delete)

**CRUD (Create, Read, Update, Delete)** representa as quatro operações básicas em sistemas de banco de dados: criar (Create), ler (Read), atualizar (Update) e excluir (Delete). É fundamental para manipulação de dados.

## 59. Big O Notation

**Big O Notation** descreve a eficiência ou complexidade de um algoritmo em termos de seu pior cenário. Ajuda os desenvolvedores a avaliar o desempenho de um algoritmo à medida que os dados aumentam.

## 60. KISS (Keep It Simple, Stupid)

**KISS (Keep It Simple, Stupid)** é um princípio de design que sugere manter as soluções simples e diretas. Evitar complexidades desnecessárias contribui para sistemas mais robustos.

## 61. MVC (Model-View-ViewModel)

**MVC (Model-View-ViewModel)** é um padrão de arquitetura de software que divide uma aplicação em três componentes principais: Model (Modelo), View (Visão) e ViewModel (Modelo de Visão). Facilita a separação de preocupações.

## 62. DSL (Digital Subscriber Line)

**DSL (Digital Subscriber Line)** refere-se a uma tecnologia de comunicação de dados que fornece conexões de internet de alta velocidade por meio de linhas telefônicas tradicionais.

## 63. DAG (Directed Acyclic Graph)

**DAG (Directed Acyclic Graph)** é um gráfico direcionado sem ciclos. É frequentemente usado para representar dependências entre tarefas ou elementos em sistemas computacionais.

## 64. Jenkins

**Jenkins** é uma ferramenta de automação de código aberto usada para automação de compilação, teste e implantação contínua. Ajuda a integrar e entregar código de maneira eficiente.

## 65. JWT (Java Web Token)

**JWT (Java Web Token)** é um padrão aberto que define uma maneira compacta e autônoma de transmitir informações entre partes como um objeto JSON. É frequentemente usado para autenticação e autorização.

## 66. JWT (Java Writing Tool)

**JWT (Java Writing Tool)** aqui se refere a ferramentas em Java usadas para criar e manipular Java Web Tokens.

## 67. OAuth

**OAuth** é um protocolo de autorização que permite que aplicativos acessem os recursos de um usuário sem revelar suas credenciais. É comumente usado para autenticação em APIs.

## 68. Sprint

**Sprint** é um período de tempo fixo em métodos ágeis, como Scrum, durante o qual uma equipe trabalha para completar um conjunto de tarefas previamente definido.

## 69. Continuous Integration (Integração Contínua)

**Continuous Integration (Integração Contínua)** é uma prática de desenvolvimento que envolve a integração frequente do código produzido por membros de uma equipe. Isso ajuda a detectar e corrigir problemas rapidamente.

## 70. Inversion of Control (IoC)

**Inversion of Control (IoC)** é um padrão de design que inverte o controle de execução de uma aplicação. Em vez de um componente chamar outro, um contêiner IoC gerencia a execução.

## 71. Thunk

**Thunk** é uma função que encapsula uma expressão para adiar sua avaliação. É comumente usado em contextos assíncronos.

## 72. JWT (JavaScript Writing Technique)

**JWT (JavaScript Writing Technique)** aqui se refere a técnicas de escrita de código em JavaScript relacionadas à manipulação de Java Web Tokens.

## 73. Docker Compose

**Docker Compose** é uma ferramenta para definir e executar aplicativos Docker multi-container. Permite que você defina a configuração de vários serviços em um único arquivo.

## 74. Thread

**Thread** é a menor unidade de execução em um sistema operacional. Em programação, refere-se a uma sequência de instruções que pode ser executada simultaneamente.

## 75. Semaphore

**Semaphore** é um conceito usado em programação concorrente para controlar o acesso a recursos compartilhados. É uma variável que controla o acesso concorrente a recursos.

## 76. Singleton Pattern

**Singleton Pattern** é um padrão de design que restringe a instanciação de uma classe a um único objeto e fornece um ponto global de acesso a ele.

## 77. Load Balancer

**Load Balancer** distribui o tráfego de rede ou as operações de computação entre vários servidores para otimizar o desempenho, maximizar a disponibilidade e evitar sobrecargas.

## 78. Mocking

**Mocking** é uma técnica de teste em que objetos simulados são usados para representar objetos do mundo real. Isso ajuda a isolar a unidade de código que está sendo testada.

## 79. Mockito

**Mockito** é uma biblioteca de teste para Java que permite a criação de objetos simulados (mocks) para testes unitários.

## 80. Jira

**Jira** é uma ferramenta de gerenciamento de projetos que suporta metodologias ágeis. É comumente usado para rastrear o progresso do desenvolvimento de software.

## 81. WebSocket

**WebSocket** é um protocolo de comunicação bidirecional, em tempo real, entre clientes e servidores. É usado para aplicativos web que exigem comunicação instantânea.

## 82. WebAssembly

**WebAssembly** é um formato de código binário que permite a execução de código de baixo nível em navegadores da web. Ele visa fornecer desempenho próximo ao código nativo.

## 83. Retrofit

**Retrofit** é uma biblioteca para Android e Java que simplifica o consumo de serviços web RESTful. Facilita a conversão de APIs HTTP em interfaces Java.

## 84. Lambda Expression

**Lambda Expression** é uma função anônima que permite tratar funções como objetos em linguagens de programação. É especialmente útil em programação funcional.

## 85. WebSockets

**WebSockets** é uma tecnologia que permite a comunicação bidirecional e em tempo real entre clientes e servidores por meio de uma única conexão persistente.

## 86. Nginx

**Nginx** é um servidor web de código aberto conhecido por sua eficiência e desempenho. É comumente usado como um servidor proxy reverso e balanceador de carga.

## 87. AJAX (Asynchronous JavaScript and XML)

**AJAX (Asynchronous JavaScript and XML)** é uma técnica de desenvolvimento web que permite atualizações de página assíncronas, sem a necessidade de recarregar toda a página.

## 88. JPA (Java Persistence API)

**JPA (Java Persistence API)** é uma especificação do Java que descreve o gerenciamento de dados relacionais em aplicativos Java. Facilita o mapeamento de objetos Java para bancos de dados relacionais.

## 89. JDBC (Java Database Connectivity)

**JDBC (Java Database Connectivity)** é uma API Java que permite a comunicação com bancos de dados relacionais. Ele fornece uma interface para executar consultas SQL e atualizar dados.

## 90. Maven

**Maven** é uma ferramenta de automação de construção usada principalmente para projetos Java. Ele gerencia dependências e automatiza o processo de construção do software.

## 91. Pandas

**Pandas** é uma biblioteca de análise de dados em Python. Ela fornece estruturas de dados e ferramentas para manipulação e análise eficientes de conjuntos de dados.

## 92. Flask

**Flask** é um framework web leve para Python. É usado para criar aplicativos web simples e rápidos, seguindo o princípio "faça o mínimo possível".

## 93. Redux

**Redux** é um padrão de gerenciamento de estado comumente usado em aplicativos React. Ele ajuda a manter o estado da aplicação de forma previsível e centralizada.

## 94. Immutable.js

**Immutable.js** é uma biblioteca JavaScript que oferece estruturas de dados imutáveis, o que significa que seus valores não podem ser alterados após a criação. Isso facilita o controle de estado em aplicações.

## 95. RabbitMQ

**RabbitMQ** é um sistema de mensageria de código aberto que facilita a comunicação entre diferentes partes de uma aplicação distribuída. Ele implementa o protocolo Advanced Message Queuing Protocol (AMQP).

## 96. MapReduce

**MapReduce** é um modelo de programação para processamento paralelo de grandes conjuntos de dados em clusters. Ele divide tarefas em etapas de mapeamento e redução.

## 97. CRUD (Create, Read, Update, Delete)

**CRUD (Create, Read, Update, Delete)** representa as quatro operações básicas em sistemas de banco de dados: criar (Create), ler (Read), atualizar (Update) e excluir (Delete). É fundamental para manipulação de dados.

## 98. MERN Stack

**MERN Stack** é uma pilha de tecnologia que combina MongoDB, Express.js, React e Node.js. É usada para desenvolver aplicativos web modernos de ponta a ponta.

## 99. JWT (Joyent's Web Token)

**JWT (Joyent's Web Token)** é um padrão aberto que define uma maneira compacta e autônoma de transmitir informações entre partes como um objeto JSON. É frequentemente usado para autenticação e autorização.

## 100. OAuth

**OAuth** é um protocolo de autorização que permite que aplicativos acessem os recursos de um usuário sem revelar suas credenciais. É comumente usado para autenticação em APIs.

## 101. Sprint

**Sprint** é um período de tempo fixo em métodos ágeis, como Scrum, durante o qual uma equipe trabalha para completar um conjunto de tarefas previamente definido.

## 102. Continuous Integration (Integração Contínua)

**Continuous Integration (Integração Contínua)** é uma prática de desenvolvimento que envolve a integração frequente do código produzido por membros de uma equipe. Isso ajuda a detectar e corrigir problemas rapidamente.

## 103. Inversion of Control (IoC)

**Inversion of Control (IoC)** é um padrão de design que inverte o controle de execução de uma aplicação. Em vez de um componente chamar outro, um contêiner IoC gerencia a execução.

## 104. Thunk

**Thunk** é uma função que encapsula uma expressão para adiar sua avaliação. É comumente usado em contextos assíncronos.

## 105. JWT (Java Writing Tool)

**JWT (Java Writing Tool)** aqui se refere a ferramentas em Java usadas para criar e manipular Java Web Tokens.

## 106. WebSocket

**WebSocket** é um protocolo de comunicação bidirecional, em tempo real, entre clientes e servidores. É usado para aplicativos web que exigem comunicação instantânea.

## 107. WebAssembly

**WebAssembly** é um formato de código binário que permite a execução de código de baixo nível em navegadores da web. Ele visa fornecer desempenho próximo ao código nativo.

## 108. Retrofit

**Retrofit** é uma biblioteca para Android e Java que simplifica o consumo de serviços web RESTful. Facilita a conversão de APIs HTTP em interfaces Java.

## 109. Lambda Expression

**Lambda Expression** é uma função anônima que permite tratar funções como objetos em linguagens de programação. É especialmente útil em programação funcional.

## 110. WebSockets

**WebSockets** é uma tecnologia que permite a comunicação bidirecional e em tempo real entre clientes e servidores por meio de uma única conexão persistente.

## 111. Docker Swarm

**Docker Swarm** é uma ferramenta que permite a criação e gerenciamento de clusters de contêineres Docker. Facilita a escalabilidade e a distribuição de aplicativos em ambientes contêinerizados.
